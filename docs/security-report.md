Cloud Security Setup Report

Introduction

This project demonstrates the implementation of IAM policies, secure storage, and data encryption on a cloud platform.

IAM Policy Configuration

An IAM user was created with restricted permissions based on the Principle of Least Privilege.

Permissions assigned:

- s3:ListBucket
- s3:GetObject

Secure Storage

Amazon S3 was used as secure cloud storage.

Security features enabled:

- Block Public Access
- Bucket Versioning
- Access Control Policies

Data Encryption

Server-side encryption was enabled using AES-256.

Benefits:

- Protects stored data
- Prevents unauthorized access
- Improves security compliance

Results

Successfully implemented:

1. IAM Policies
2. Secure Storage
3. Data Encryption

Conclusion

Cloud resources were secured using IAM access controls, secure storage settings, and encryption mechanisms.
